# LogicPilot

> A front-end-first RTL / FPGA / ASIC design-flow agent plugin for Claude Code and Codex.

**License:** MIT | **Python:** 3.10+ (3.10 needs `pip install tomli`) | **Platforms:** Linux · macOS · Windows

---

## Introduction

LogicPilot is not an RTL generator and is not tied to any EDA vendor.

It wraps the error-prone parts of the hardware front-end flow — design planning, static checks, simulation, synthesis, CDC, and power — into agent-readable JSON contracts so Claude Code / Codex can drive the flow like CI. Every stage result is generated by a Python driver; the AI reads the output and cannot declare a pass on its own.

**Not included:** DFT/ATPG, back-end place-and-route (PnR is a passthrough), IP integration, GDS sign-off.

---

## Flow

```
Spec → Micro-arch → RTL → Source audit → TB audit → Lint → Sim → Synth → [Backend] → Report
         ↑                     ↑              ↑              ↑           ↑
  hardware-design-planning   audit         tb-audit        sim         synth
```

Default front-end pipeline (`/lp-front`):

```
plan-check → audit → tb-audit → lint → sim → synth → report
```

Each stage outputs JSON. `status` is one of `pass / fail / blocked / skip / timeout`. The pipeline halts at the first non-pass stage and annotates the reason.

## Features

| Feature | Command | Notes |
|---|---|---|
| Project init | `/lp-init` | Generates `flow.toml` and planning doc templates |
| Environment check | `/lp-doctor` | Checks Python version, config, tool availability |
| Front-end pipeline | `/lp-front` | Chains plan-check → audit → tb-audit → lint → sim → synth → report |
| RTL static audit | `/lp-audit` | No EDA required; flags latches, multi-drivers, non-synthesizable constructs |
| TB audit | `/lp-tb` | Flags wave-only tests, missing PASS/FAIL, unseeded randomization, etc. |
| Lint | `/lp-lint` | Dispatches to verilator / verible / ghdl |
| Simulation | `/lp-sim` | Dispatches to verilator / iverilog / ghdl / questa etc. |
| Synthesis | `/lp-synth` | Dispatches to yosys / vivado / quartus / openroad etc. |
| CDC check | `/lp-cdc-check` | SpyGlass CDC primary, verilator `--cdc` fallback; auto-skip for single-clock designs |
| Formal verification | `/lp-formal` | Dispatches to sby / jaspergold / vcf / qverify |
| Power analysis | `/lp-power` | VCD → SAIF → power report; always reports activity assumptions |
| Constraint generation | `/lp-constraints` | Generates SDC/XDC constraint templates |
| Synth report reader | `synth-report-reader` sub-agent | Parses large synth logs; extracts WNS/TNS/utilization/structural warnings |
| Result triage | `logicpilot-result-triage` skill | Summarizes JSON output as verdict → cause → next action |

**Hard gates (must pass before declaring done):**

- **Planning gate:** New modules must pass `plan-check` before any RTL is written.
- **CDC gate:** Designs with ≥2 unrelated clocks cannot declare done until CDC check passes.
- **Discipline gate:** Apply `hardware-design-discipline` before every RTL edit.

## Skills

| Skill | Purpose |
|---|---|
| `hardware-design-planning` | Spec, micro-architecture, task breakdown |
| `hardware-design-discipline` | Assumption surfacing and goal definition before RTL edits |
| `hardware-rtl-design` | RTL authoring and review |
| `hardware-rtl-audit` | Source risk scan (no EDA dependency) |
| `hardware-synthesizable-coding` | Synthesizable coding rules |
| `hardware-reset-design` | Reset architecture and reset-domain crossing |
| `hardware-fsm-design` | FSM design and encoding |
| `hardware-cdc` | Clock-domain and reset-domain crossing structural review |
| `hardware-constraints` | SDC/XDC timing constraints |
| `hardware-interfaces` | AXI / APB / AHB / Wishbone and other bus protocols |
| `hardware-simulation` | Simulation and debug |
| `hardware-verification` | Verification planning, TB architecture, coverage |
| `hardware-synthesis` | Synthesis report interpretation |
| `hardware-power-analysis` | Power estimation and budgeting |
| `systemverilog-design-modeling` | SV packages / interfaces / typedefs |
| `systemverilog-verification-platform` | SV testbench platform architecture |
| `fpga-architecture-optimization` | RTL-level FPGA optimization |
| `fpga-timing-closure` | Post-synth/post-PnR timing closure iteration |
| `logicpilot-result-triage` | LogicPilot JSON result classification |

## Installation

### Claude Code

```
/plugin marketplace add shijhtop/LogicPilot
/plugin install logicpilot@logicpilot-marketplace
```

### Codex

Open `/plugins` → **Add marketplace** → enter `shijhtop/LogicPilot` → install **LogicPilot**.

## Development

`shared/flow` is the single source for the flow driver. `shared/skills` is the single source for skills. `claude-code/plugins/logicpilot/` is the generated platform package (synced from `shared/`).

python3 -m pytest shared/flow/tests -q
python3 -m pytest shared/skill_tests -q
